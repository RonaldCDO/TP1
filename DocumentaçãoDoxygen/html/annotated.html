Documentação de Domínios e Entidades para Sistema de Caronas

Lista de componentes
Lista de classes, estruturas, uniões e interfaces com uma breve descrição:
 CAssentoÉ a posição do banco utilizado pelo passageiro em um veículo
 CBagagemSão os objetos de uso pessoal do usuário, os quais são transportados de forma empacotada em malas, mochilas, sacolas, etc
 CCaronaA Entidade Carona é uma abstração para o evento principal do sistema de carona, o qual é indentificada pelos objetos codigo, cidadeDeOrigem, estadoDeOrigem, cidadeDeDestino, estadoDeDestino, data, duracao, vagas, preco
 CCidadeÉ um aglomerado geográfico de pessoas que desempenha funções socioeconômicas
 CCodigoDeBancoDomínio que identifica a entidade bancária cujo o usuário do sistema é cliente
 CCodigoDeCaronaNúmero utilizado para identificar uma carona composta por diferentes usuários do sistema
 CCodigoDeReservaNúmero utilizado pelo sistema e pelos para identificar o planejamento e previsão do evento no domínio de aplicação
 CContaA Entidade Conta é uma abstração para a escrituração de registro individual na instituição financeira do usuário do sistema, a qual é indentificada pelos objetos codigoDeBanco, numeroDeAgencia e numeroDeConta
 CCPFCPF é a sigla para Cadastro de Pessoa Física
 CDataIndicação do dia, mês e ano em que determinado fato ocorreu ou deverá ocorrer
 CDominioA Classe Dominio é uma classe base para os domínios, que são elementos básicos de um sistema de caronas
 CDuracaoEspaço de tempo no qual o evento do domínio de aplicação acontece
 CEmailSistema de intercâmbio de mensagens entre computadores ligados em rede utilizado para contactar os usuários
 CEstadoCada um dos territórios autônomos dentro de uma federação
 CNomePalavra dada ao usuário que compõe um dos elementos básicos da sua individualização, além de ser um dos identificadores no sistema
 CNumeroDeAgenciaDado o código de banco que identifica a entidade bancária do usuário, o número de agência identifica o local onde tal agente se estabelece e o usuário possui seu cadastro
 CNumeroDeContaÉ um número que identifica a escrituração de registro individual do usuário do sistema em sua instituição financeira
 CPrecoValor econômico de um serviço ou mercadoria
 CReservaA Entidade Reserva é uma abstração para o registro de um usuário em uma carona do sistema, a qual é indentificada por objetos das classe Codigo, Assento e Bagagem
 CSenhaCombinação de caracteres responsável pela segurança da conta e pela identificação do usuário
 CTelefoneNúmeros por meio dos quais se estabelece uma conexão telefônica, que são utilizados para contactar o usuário
 CUsuarioA Entidade Usuario é uma abstração para o usuário do sistema de carona, o qual é indentificado pelos objetos das classes Nome, Telefone, Email, Senha e CPF
 CVagasEspaços disponíveis para incorporação de novos usuários
